This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a **business case / initiative project** (not a software codebase) for introducing medical-grade Hilotherm cooling equipment at Sanctura, a private oncology clinic in Fitzrovia, London. The initiative covers CIPN prevention (hands/feet cooling) and scalp cooling (hair loss prevention) during chemotherapy.

**Proposed by:** Maria Vega (Nurse at Sanctura)
**Key supporter:** Prof. Justin Stebbing (Referring Oncologist)
**Owner:** Grant (presenting to the board)

## Repository Structure

- `background/` - Original source documents: Maria's executive summary, clinical research papers (PDFs), Hilotherm price list, invoice. **Do not modify these files.**
- `information-base/` - Living reference documents (01-09 series) synthesised from background materials. These are the primary knowledge base to consult when answering questions or drafting documents.
- `output/` - Draft communications, proposals, and board papers. Files are prefixed `yyyy-dd-mm-`.
- `new-information/` - Drop zone for new documents to be processed and integrated into the information base.

## Working With This Project

- When asked to produce documents (emails, proposals, board papers), draft them in `output/` with a `yyyy-dd-mm-` date prefix.
- When new files appear in `new-information/`, integrate their content into the relevant `information-base/` documents and note any new action items or risks.
- Always consult `information-base/` files before answering questions - they contain the synthesised knowledge from all background materials.
- The `background/` folder is read-only reference material. Never modify these files.
