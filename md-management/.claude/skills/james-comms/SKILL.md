# James's Communication Profile
- Extremely fast thinker and analytical
- Prefers facts over narrative
- Consumes information very fast
- Dislikes waffle, preamble, or unnecessary context
- Will scan the top-level summary and only drill into detail if needed

## Communication Structure

### Simple Communications (short updates, confirmations, single-point responses)
Use summary only — no elaboration needed.

```
[One-line context/subject if not obvious from email thread]

[Distilled facts / raw decision points / proposed action]
- Bullet format preferred
- Each bullet = one fact or decision
- No filler words

[Sign-off]
```

### Complex Communications (multi-point feedback, proposals, updates with several items)
Lead with distilled summary, then optional elaboration below the fold.

```
[One-line context/subject if not obvious from email thread]

**Summary:**
- [Fact/decision 1]
- [Fact/decision 2]
- [Fact/decision 3]

[Sign-off]

---

**Detail (if needed):**

[Elaboration on point 1 — assume James may not read this, but it's there if he wants to drill in]

[Elaboration on point 2]
```

## Style Rules
1. **Lead with the answer/decision** — not the reasoning
2. **Bullet points over paragraphs** — always
3. **Bold key labels** to enable fast scanning (e.g., **Updated:**, **No change:**, **Action needed:**)
4. **Group by action type** where possible (e.g., "Updated in job spec" / "No change needed" / "Noted separately")
5. **No pleasantries beyond a single short opener** (e.g., "Thanks for the review." not "Thank you so much for taking the time to review this thoroughly, really appreciate it.")
6. **No hedging language** — state positions clearly
7. **Use "I" not "we"** unless genuinely referring to a group decision
8. **Keep sign-off minimal** — just the name, no "Kind regards" or "Best wishes"
