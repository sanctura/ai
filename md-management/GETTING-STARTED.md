# Getting Started Guide

## Initial Setup (Do This First)

Your management system is now structured and ready to use. Here's how to get started:

### Step 1: Fill in Business Context (30-60 minutes)

These foundational files establish the context for all future work:

1. **`business/overview.md`**
   - Company basics, mission, vision
   - Strategic priorities for this year
   - Key targets and metrics
   - Market context and constraints

2. **`business/organization.md`**
   - Your org chart and reporting structure
   - Your dual role (MD/CTO) breakdown
   - Team composition and headcount
   - Decision-making framework

3. **`business/ceo-directives.md`**
   - Current strategic guidance from CEO
   - Recent feedback and action items
   - Areas requiring alignment

### Step 2: Document Your Team (30-45 minutes)

4. **`people/team-roster.md`**
   - All direct reports with key details
   - Extended team overview
   - 1:1 schedule
   - Team health indicators

5. **`people/hiring.md`**
   - Open positions and their status
   - Recruiting pipeline
   - Future hiring needs

6. **Create individual performance files**
   - Copy `people/performance/TEMPLATE.md` for each direct report
   - Name as `people/performance/[firstname-lastname].md`
   - Fill in current status and recent 1:1 notes

### Step 3: Capture Active Work (20-30 minutes)

7. **`projects/active.md`**
   - List all current projects
   - Status, health, owners
   - Key milestones and blockers

8. **`projects/pipeline.md`**
   - Upcoming projects
   - Ideas under evaluation
   - Backlog items

### Step 4: Document Risks & Opportunities (15-20 minutes)

9. **`tracking/risks.md`**
   - Current risks you're monitoring
   - Mitigation plans

10. **`tracking/opportunities.md`**
    - Strategic opportunities being evaluated
    - Potential upside and requirements

### Step 5: Set Up Metrics Dashboard (15 minutes)

11. **`tracking/metrics.md`**
    - Fill in your key metrics
    - Add current values and targets
    - Note trends

### Step 6: Plan Your Current Week (15 minutes)

12. **`planning/current-week.md`**
    - Top 3 priorities for this week
    - Key meetings and calendar
    - Decisions needed
    - Focus time blocks

---

## How to Use This System

### Daily
- Review `planning/current-week.md` each morning
- Update project status as things change
- Note decisions in `planning/decisions.md`
- Capture 1:1 notes in individual performance files

### Weekly
- **Monday**: Create new `planning/current-week.md` (copy template)
  - Archive last week's to `planning/archive/`
  - Set top 3 priorities
  - Block focus time
- **Friday**: Complete end-of-week review section
  - What got done, what didn't
  - Key learnings
  - Adjustments needed

### Monthly
- Review and update `tracking/metrics.md`
- Update all `people/performance/` files before 1:1s
- Review `tracking/risks.md` and `tracking/opportunities.md`
- Check alignment with `business/ceo-directives.md`

### Quarterly
- Update strategic sections in `business/overview.md`
- Review and adjust `projects/pipeline.md`
- Conduct thorough performance reviews (update all performance files)
- Update org structure if changed

### Ad Hoc
- **When CEO gives feedback**: Update `business/ceo-directives.md`
- **When projects change status**: Update `projects/active.md`
- **When new risks emerge**: Add to `tracking/risks.md`
- **When making key decisions**: Document in `planning/decisions.md`
- **After 1:1s**: Update notes in relevant performance files

## Tips for Success

1. **Be honest and detailed** - The more accurate your input, the better guidance you'll receive
2. **Update regularly** - Stale data leads to bad decisions
3. **Use git commits** - Track changes over time to see patterns
4. **Keep it current** - Better to update often with small changes than rarely with big updates
5. **Trust the system** - Consistently using it will make it more valuable over time
